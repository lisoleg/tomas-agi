## 修复知识列表/语料列表bug

### 问题
1. 概念列表和关系列表点击无效 — 后端 API 返回字段名与前端 KnowledgeItem 接口不匹配
2. 蒸馏后语料列表不更新 — 同样是 API 格式问题

### 根本原因
- 后端 `knowledge_items` 表没有 `type` 列，POST 时丢失 `type` 信息
- GET `/api/knowledge` 返回 `{concept, content, source}` 而非前端期望的 `{type, label, extra, domain, createdAt}`
- `knowledge_triples` 表为空 → `/api/knowledge/graph` 返回空 → 页面刷新后 graphData=null → 所有点击失效

### 修复内容
1. `server.py`: `knowledge_items` 表加 `type TEXT DEFAULT 'concept'` 列（含迁移）
2. `server.py`: GET `/api/knowledge` 返回前端期望格式（type/label/extra/domain/createdAt）
3. `server.py`: POST `/api/knowledge` 保存 `type` 和 `createdAt`，关系自动写入 `knowledge_triples`
4. `DistillPanel.tsx`: 页面加载时若无三元组数据，仍构建最小 graphData（仅顶点）使点击可用

### 后端 API 修复后格式
```json
{
  "id": 3,
  "type": "concept",
  "label": "尼古拉·哥白尼",
  "extra": "𝕀=0.900",
  "domain": "蒸馏",
  "createdAt": 1781372246000
}
```

## 修复概念提取伪概念过滤（日期被当概念）

### 问题
LLM 蒸馏返回的"1473年2月19日"、"1543年5月24日"等日期被当作概念入库并显示在概念列表中。

### 根本原因
`DistillPanel.tsx` 保存知识条目时（第413行 `concepts.map(...)`）**全量保存了 LLM 返回的所有概念，未经过 `isPseudoConcept` 过滤**。虽然前端有 `filteredDistilledConcepts` 用于蒸馏结果展示区过滤，但持久化存储区（来自后端 DB）的数据入库时完全没过滤。

### 修复内容
1. **`DistillPanel.tsx` 核心修复**: 保存前先 `concepts.filter(c => !isPseudoConcept(c.concept).isPseudo)`，用过滤后的 `filteredConcepts` 入库和构建 graphData
2. **`DistillPanel.tsx` 加强 `isPseudoConcept`**: 新增月日格式、日期范围/生卒（1473—1543）、纯四位数字年份等正则
3. **`llm_distiller.py` 同步对齐**: 后端 `_DATE_PATTERNS` 和 `is_pseudo_concept` 同步加强相同规则
4. **清理脏数据**: 删除 DB 中已入库的 `id=11(1473年2月19日)` 和 `id=12(1543年5月24日)`

## 参考 OwnThink 模型重构蒸馏 prompt

### 分析
研究了 https://github.com/ownthink/KnowledgeGraphData 的知识图谱模型：
- OwnThink 用 `(实体, 属性, 值)` 三元组，每个实体必须有"描述"属性
- 核心启发：**能写出有意义描述的才是实体**

### 改进方案
1. **前端 `distiller.ts`**：重构 `extractConcepts` prompt
   - 输出格式从 `{concept, importance, context}` 改为 `{entity, description, tags, importance}`
   - 加入 ✅ 正确示例和 ❌ 错误示例 few-shot
   - 加 "describe or discard" 后置校验：描述空洞（"一个XX"、不足5字、等于实体名）直接丢弃
   - 兼容新旧两种 LLM 输出格式
2. **后端 `llm_distiller.py`**：同步重构 `extract_concepts` 方法
   - 相同的 prompt 设计 + few-shot + describe-or-discard
   - 后端 TRIVIAL_DESC_PATTERNS 过滤描述空洞的伪实体
3. **核心原则**：「写不出有意义的定义 → 就不要返回」— 从源头杜绝日期/数字混入概念列表

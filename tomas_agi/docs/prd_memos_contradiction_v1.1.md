## 1. 背景

当前 `memos_fusion.py` 中的 `_is_contradictory()` 函数是简化版，基于字符级比较检测矛盾。文章提到 MUS 双存机制需要精确的矛盾检测，因此需要升级。

---

## 2. 增量需求（三个改进方向）

### 2.1 否定词检测（优先级：P0 - 立即实现）

**目标**：检测否定词（"不是"、"不可能"、"错误"），区分肯定/否定陈述。

**示例**：
- "心主神明" vs "心不主神明" → 矛盾
- "太阳绕地球转" vs "太阳不绕地球转" → 矛盾

**实现方案**：
- 构建否定词词典：`["不", "不是", "不可能", "错误", "不对", "否认"]`
- 检测逻辑：如果两个关系包含不同的否定词组合 → 矛盾

---

### 2.2 NLP 主语-谓语-宾语提取（优先级：P1 - 第二版实现）

**目标**：使用 NLP 工具提取关系的主语、谓语、宾语，精确检测矛盾。

**示例**：
- "心主神明" → 主语：心，谓语：主，宾语：神明
- "脑主神明" → 主语：脑，谓语：主，宾语：神明
- 矛盾检测：主语不同（心 vs 脑），但谓语+宾语相同 → 矛盾

**实现方案**：
- 使用 **jieba + HanLP** 或 **LAC（百度）** 进行中文分词和依存句法分析
- 提取主谓宾三元组
- 矛盾判定规则：
  - 如果主语不同，但谓语+宾语相同 → 矛盾
  - 如果主语+宾语相同，但谓语相反（"主" vs "不主"） → 矛盾

**技术方案**：
- **方案 A**（推荐）：使用 `lac` 库（百度 LAC，轻量级）
  ```python
  from LAC import LAC
  lac = LAC(mode='seg')
  # 或 lac = LAC(mode='rank')
  ```
- **方案 B**：使用 `jieba` + 规则提取（简化版，不依赖外部库）

---

### 2.3 EML 语义相似度计算（优先级：P2 - 第三版实现）

**目标**：接入 EML 知识库，计算概念的语义相似度，检测隐含矛盾。

**示例**：
- "心主神明" 和 "脑主神明" 在 EML 图中可能都连接到 "神明"
- 但如果 EML 图显示 "心" 和 "脑" 是互斥概念（asym ≠ 0） → 矛盾

**实现方案**：
- 查询 EML 图（`EMLFileLoader` 或 `hyperedge.py`）
- 计算概念相似度：
  - 如果两个概念在 EML 图中直接相连 → 相似度高
  - 如果两个概念在 EML 图中 asym ≠ 0 → 矛盾
- 矛盾判定：如果 `build_eml_edge(concept1, concept2)` 返回 `asym ≠ 0` → 矛盾

**依赖**：
- 需要加载 EML 文件（`.eml` 和 `.concepts.json`）
- 需要 `EMLFileLoader` 或 `HypEdge` 类
